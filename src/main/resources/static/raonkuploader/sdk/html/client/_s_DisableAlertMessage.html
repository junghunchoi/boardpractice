Script 설정이 우선 적용됩니다.
업로드 모든 confirm창을 표시하거나 숨기는 기능을 설정합니다.
각 상황별 alert의 활성화/비활성화 를 설정합니다.

remarks

disable_delete_confirm  기본값은 "0" 이고 "모든 confirm창 활성", "1"로 설정시 "모든 confirm창 비활성"입니다.
"2"로 설정시 "선택 항목제거만 confirm창 비활성"이고,
"3"로 설정시 "전체 항목제거만 confirm창 비활성"입니다.

duplication  "파일 추가 중복 시" alert창 비활성합니다.
기본값은 "1" 이고, "0"으로 설정시 파일 추가 중복 시 alert창 활성합니다.

delete_unchosen  "파일 삭제 시" 선택된 파일이 없을 경우 alert창 비활성합니다.
기본값은 "1" 이고, "0"으로 설정시 파일 삭제시 선택된 파일이 없을 경우 alert창 활성합니다.

download_unchosen  "파일 다운로드 시" 선택된 파일이 없을 경우 alert창 비활성합니다.
기본값은 "1" 이고, "0"으로 설정시 파일 삭제시 선택된 파일이 없을 경우 alert창 활성합니다.

open_unchosen  "파일 열기 시" 선택된 파일이 없을 경우 alert창 비활성합니다.
기본값은 "1" 이고, "0"으로 설정시 파일 삭제시 선택된 파일이 없을 경우 alert창 활성합니다.

file_extension_detect  "확장자 변조 파일 추가시" 선택된 파일의 확장자가 변조된 경우 alert창 비활성합니다.
기본값은 "1" 이고, "0"으로 설정시 선택된 파일의 확장자가 변조된 경우 alert창 활성합니다.

file_max_count_add  "파일 추가 버튼 클릭 시" 파일 최대 개수를 초과할 경우 alert창 비활성합니다.
기본값은 "1" 이고, "0"으로 설정시 파일 최대 개수를 초과할 경우 alert창 활성합니다.

disable_upload_cancel_confirm  업로드 시 전송 취소를 할 경우 confirm창을 비활성합니다.
기본값은 "0" 이고, "1"로 설정시 비활성화 됩니다.

disable_recovery_confirm  자동 복구를 할 경우 confirm창을 비활성합니다.
기본값은 "0" 이고, "1"로 설정시 비활성화 됩니다.
